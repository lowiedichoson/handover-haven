# Glossary

A quick reference for the acronyms and terms you'll see throughout the handovers.
If you run into something not listed here, add it.

## Companies

| Acronym | Full Name | What They Do |
|---|---|---|
| **PEMI** | Philequity Management, Inc. | Asset management — runs mutual funds and managed portfolios. |
| **VEI** | Vantage Equities, Inc. | Stock brokerage — equity trading for retail and institutional clients. |
| **VFC** | Vantage Financial Corp. | Financial services — back-office operations, settlement, treasury, and branch banking. |

## Teams

| Acronym | Full Name |
|---|---|
| **ICTG** | Information & Communications Technology Group |

## Systems

| Name / Acronym | What It Is |
|---|---|
| **Clientease** | Client management platform (frontend + backend API). Built in-house by ICTG. |
| **Clientease Backend** | REST API and database for Clientease. |
| **Clientease Frontend** | Web UI for Clientease. |
| **eSettlement** | Back-office settlement system used by VFC. Handles trade settlement, bridge entries, and accounting feeds. |
| **eTerminal** | Branch terminal system used by VFC branches for transaction processing. |
| **FTR Generator** | Fixed-Term Report generator — produces regulatory or internal reports. |
| **LOI Generator** | Letter of Instruction generator — used by PEMI for client instructions. |
| **Navision** | Microsoft Dynamics NAV — ERP used by VFC for accounting and financial management. |
| **Production Archiving** | System or process for archiving production data and logs. |
| **Synapse** | Integration/middleware system connecting various platforms. |
| **Voyager** | Trading or order management system. |
| **Voyager FX** | Foreign exchange module or variant of Voyager. |

## Common Terms

| Term | Meaning |
|---|---|
| **Bridge Entry** | A manual journal entry used to reconcile or transfer data between systems (e.g., eSettlement → Navision). |
| **Backend Formula** | Server-side calculation logic — often in eSettlement for trade computations, fees, or taxes. |
| **Production Support** | Day-to-day operational support — fixing incidents, running reports, assisting users. |
| **Runbook** | A step-by-step guide for resolving a specific incident or completing a recurring task. |
